<task type="auto">
  <name>Task 2: Create data/README.md data dictionary</name>
  <files>data/README.md</files>
  <action>
Create data/README.md as the canonical data dictionary. The file must contain:

**1. Header and purpose statement:**
Brief description that this directory stores European-style index option price data for SSVI calibration testing.

**2. Directory Structure section:**
Document the source-first hierarchy pattern: `data/{source}/{underlying}/{YYYY-MM-DD}.csv`. Explain that each CSV contains the complete option chain for one underlying on one trading date. List current source directories (cboe/spx, eurex/sx5e, sample).

**3. File Naming Convention section:**
Files are named `YYYY-MM-DD.csv` using ISO 8601 date format. The date is the observation/quote date. Example: `data/cboe/spx/2024-01-19.csv`.

**4. Canonical CSV Schema section with Column Definitions table:**

Required columns (all must be present in every data file):

| Column | Type | Unit | Required | Description |
|--------|------|------|----------|-------------|
| quote_date | date | YYYY-MM-DD | Yes | Observation/trading date |
| expiry | date | YYYY-MM-DD | Yes | Option expiration date |
| strike | float | Index points | Yes | Strike price |
| option_type | string | C or P | Yes | Call (C) or Put (P) |
| bid | float | Index points | Yes | Best bid price |
| ask | float | Index points | Yes | Best ask price |
| underlying_price | float | Index points | Yes | Spot/index level at quote time |

Preferred columns (include if source provides; do not fabricate):

| Column | Type | Unit | Description |
|--------|------|------|-------------|
| forward | float | Index points | Forward price for this expiry |
| discount_factor | float | Dimensionless | Discount factor for this expiry |

Optional columns (include if source provides):

| Column | Type | Unit | Description |
|--------|------|------|-------------|
| volume | integer | Contracts | Trading volume |
| open_interest | integer | Contracts | Open interest |
| implied_vol | float | Annualized decimal | Source-computed implied volatility |

**5. Example CSV snippet:**
```csv
quote_date,expiry,strike,option_type,bid,ask,underlying_price,forward,discount_factor
2024-01-19,2024-02-16,4700.0,C,128.50,130.20,4780.0,4782.35,0.9987
2024-01-19,2024-02-16,4700.0,P,47.30,48.80,4780.0,4782.35,0.9987
2024-01-19,2024-02-16,4750.0,C,93.10,94.70,4780.0,4782.35,0.9987
```

**6. Design Rationale section:**
- Raw bid/ask over mid-price: preserves liquidity information for future weighting
- Raw prices over pre-computed IV: enables testing full pipeline (prices -> IV via lets_be_rational -> log-moneyness -> total variance -> CalibrationInput -> calibration)
- ISO 8601 dates: unambiguous, lexicographically sortable
- option_type as C/P: maps directly to existing q: i32 (+1/-1) convention

**7. What NOT to Store section:**
- Pre-computed implied volatility as primary data (store as optional cross-validation column only)
- Log-moneyness or total variance (derived quantities computed by future parser)
- Greeks (recomputed via black76 module)
- Filtered subsets (store full chain; filtering happens in parser)

**8. Sources section:**
Placeholder sections for CBOE DataShop and Eurex with brief descriptions. Note that specific column mappings will be added in Phase 10 when data is acquired.

**9. Data Quality Notes section:**
Placeholder for documenting known issues per data file. Format: `{source}/{underlying}/{date}: {issue description}`.

Use plain markdown. No code fences around the entire file. Keep it professional and concise.
