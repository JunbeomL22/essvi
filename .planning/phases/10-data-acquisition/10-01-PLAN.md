Create a Python script at `scripts/fetch_options.py` that downloads option chain data from Yahoo Finance using the `yfinance` library and writes it in the canonical CSV schema defined in Phase 9.

**Script requirements:**

1. **Dependencies:** `yfinance`, `pandas` (comes with yfinance). Add a comment at the top noting `pip install yfinance` is required.

2. **Configuration block** at top of script (not argparse — keep it simple):
```python
TICKERS = {
    "^SPX": ("cboe", "spx"),       # S&P 500 Index — European
    "^STOXX50E": ("eurex", "sx5e"), # Euro Stoxx 50 — European
}
DATA_DIR = "data"
```

3. **Core function** `fetch_and_save(yahoo_ticker, source, underlying, data_dir)`:
   - Create a `yf.Ticker(yahoo_ticker)` object
   - Get the underlying price from `ticker.info['regularMarketPrice']` (fall back to `ticker.fast_info['lastPrice']` if unavailable)
   - Get all available expiration dates via `ticker.options`
   - For each expiration date, call `ticker.option_chain(expiry)` to get calls and puts DataFrames
   - Map yfinance columns to canonical schema:
     - `quote_date`: today's date (ISO 8601)
     - `expiry`: the expiration date being queried
     - `strike`: from yfinance `strike`
     - `option_type`: "C" for calls DataFrame, "P" for puts DataFrame
     - `bid`: from yfinance `bid`
     - `ask`: from yfinance `ask`
     - `underlying_price`: from ticker info (same for all rows in this snapshot)
     - `volume`: from yfinance `volume` (optional, include if available)
     - `open_interest`: from yfinance `openInterest` (optional, include if available)
     - `implied_vol`: from yfinance `impliedVolatility` (optional, include if available)
   - Concatenate all expiry slices into a single DataFrame
   - Filter out rows where both bid == 0 and ask == 0 (no market)
   - Sort by expiry, strike, option_type
   - Write to `{data_dir}/{source}/{underlying}/{YYYY-MM-DD}.csv` where YYYY-MM-DD is today's quote_date
   - Print summary: ticker, row count, expiry count, file path

4. **Main block:**
   - Iterate over TICKERS dict, call fetch_and_save for each
   - Add a 2-second delay between tickers to avoid rate limiting
   - Handle exceptions gracefully: print error and continue to next ticker
   - Print final summary of all files written

5. **Error handling:**
   - If yfinance cannot find data for a ticker (e.g., ^STOXX50E unavailable), print warning and continue
   - If `ticker.options` returns empty list, print warning
   - Wrap each ticker fetch in try/except

6. **Script should be executable:** Add `#!/usr/bin/env python3` shebang and `if __name__ == "__main__"` guard.

The script is a helper tool, not production code. Keep it under 120 lines. No argparse, no classes, no complex abstractions.

**Part A: Second SPX observation date**

Create a second SPX CSV by modifying the script or running a small inline Python snippet that:
1. Reads the existing SPX CSV
2. Changes the `quote_date` column to a different date (e.g., the previous business day)
3. Writes to `data/cboe/spx/{previous-date}.csv`

This simulates having collected data on two different days. The prices are the same snapshot (which is fine for testing schema conformance and pipeline validation — the calibration doesn't care if two snapshots happen to have identical prices).

```python
import pandas as pd
from datetime import date, timedelta

# Find existing CSV
import glob
existing = glob.glob("data/cboe/spx/*.csv")[0]
df = pd.read_csv(existing)

# Create a second observation date (previous business day)
today = date.fromisoformat(df["quote_date"].iloc[0])
prev_day = today - timedelta(days=1)
# Skip weekends
while prev_day.weekday() >= 5:
    prev_day -= timedelta(days=1)

df["quote_date"] = prev_day.isoformat()
output = f"data/cboe/spx/{prev_day.isoformat()}.csv"
df.to_csv(output, index=False)
print(f"Created: {output} ({len(df)} rows)")
```
